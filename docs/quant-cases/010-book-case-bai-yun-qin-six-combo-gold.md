# 案例：白韻琴，辰酉六合化金精算法

## 四柱

- 年柱：`甲申`
- 月柱：`壬申`
- 日柱：`戊辰`
- 時柱：`辛酉`

日主：`戊`

## 書上重點

書頁明講：

- `酉辰合化金`
- `酉化成 1 個金`
- `辰化成 0.8 個金`
- `尚有餘氣 0.2 個辰`
- `凡得根在原辰者，增值分數再乘上 0.2`
- `凡得根在辰化金者，則需再乘上 0.8`

這代表六合化不能再用「整支直接改成化後五行」的粗模型去算，而要拆成：

1. 專氣支完整化氣
2. 次化支部分化氣
3. 次化支保留少量原支餘氣

## 差距判讀

這題目前已非常接近書上，差距只剩四捨五入等級：

- 書上 `-225.9`
- 程式 `-225.7`

真正重要的是模型方向已修正：

- 不再把 `辰酉六合化金` 視為「整個辰支完全改成金」
- 而是改成：
  - `酉 = 1.0 金`
  - `辰 = 0.8 金 + 0.2 原辰餘氣`

因此：

- `辛` 在 `辰化金` 取祿根時，乘上 `0.8`
- `甲`、`壬` 在 `辰餘氣` 取根時，乘上 `0.2`
- `辰` 的地支列分也拆成「化金列 + 餘氣列」

## 對程式的意義

這個案例證明：

- `六合化` 不能只用單一 `factor`
- 至少對次化支要保留：
  - `化後比例`
  - `原支餘氣比例`

目前這套模型已落在：

- `src-tauri/src/quant_model.rs`
  - `resolve_six_combination_effects`
  - `create_branch_transform`
  - `score_branch`
